# 数据源配置

LtdView 支持两种数据源类型：静态数据和 API 接口。

## 静态数据

在属性面板的「数据」标签中，选择「静态数据」类型，直接输入 JSON 格式数据。

### 数据格式

图表组件使用 name/value 格式：

```json
[
  { "name": "华东区", "value": 125000 },
  { "name": "华北区", "value": 89000 },
  { "name": "华南区", "value": 156000 }
]
```

表格组件使用对象格式：

```json
[
  { "region": "华东区", "product": "电子产品", "sales": 125000, "count": 520 },
  { "region": "华北区", "product": "食品", "sales": 89000, "count": 380 }
]
```

### 操作

- **格式化**：自动格式化 JSON 缩进
- **加载示例**：加载预设的示例数据
- **错误提示**：JSON 格式错误时显示红色提示

## API 数据源

选择「API 接口」类型，配置远程数据接口。

### 配置项

| 配置 | 说明 | 示例 |
|------|------|------|
| URL | 接口地址 | `https://api.example.com/data` |
| 方法 | HTTP 方法 | GET / POST / PUT / DELETE |
| 响应路径 | 从响应中提取数据的路径 | `data.list` |
| 请求头 | JSON 格式的自定义请求头 | `{"Authorization": "Bearer xxx"}` |
| 刷新间隔 | 自动刷新间隔（毫秒，0=不刷新） | 5000 |

### 响应路径

当 API 返回嵌套数据时，使用 `.` 分隔的路径提取：

```json
{
  "code": 200,
  "data": {
    "list": [
      { "name": "A", "value": 100 }
    ]
  }
}
```

响应路径：`data.list`

### 测试接口

点击「测试接口」按钮，验证 API 配置是否正确，显示返回数据条数。

## 字段映射

字段映射将原始数据转换为组件可用的格式。

### 映射字段

| 字段 | 说明 | 用途 |
|------|------|------|
| 名称字段 | 数据中的名称/分类字段 | 图表 X 轴 / 饼图名称 |
| 数值字段 | 数据中的数值字段 | 图表 Y 轴 / 饼图数值 |
| 系列字段 | 数据中的系列/分组字段 | 多系列图表 |

### 映射示例

原始数据：
```json
[
  { "region": "华东", "amount": 125000, "category": "电子产品" }
]
```

映射配置：
```
名称字段：region
数值字段：amount
系列字段：category
```

映射后：
```json
[
  { "name": "华东", "value": 125000, "category": "电子产品" }
]
```

## 数据刷新

### 手动刷新

修改静态数据或 API 配置后，数据自动更新。

### 自动刷新

设置 API 的「刷新间隔」（毫秒），组件会定时自动请求最新数据。

### 表单触发刷新

配置表单联动后，表单字段值变化会触发目标组件重新获取/过滤数据。

## 数据流

```
数据源配置
├── 静态数据 → JSON 编辑器输入
├── API 数据 → 配置 URL、Method、Headers、Params
├── 数据映射 → 将接口字段映射到图表维度/指标
└── 联动机制
    ├── 表单值变化 → 触发目标组件重新请求数据
    └── 下钻点击 → 携带参数请求下级数据
```
